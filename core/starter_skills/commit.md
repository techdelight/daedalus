# Commit Changes

Review staged and unstaged changes, then create a well-structured git commit.

## Instructions

1. Run `git status` and `git diff` to understand the current changes.
2. Stage relevant files (prefer specific files over `git add -A`).
3. Write a commit message following the project's convention:
   - Start with the branch/plan name and a colon
   - Summarize what changed and why
   - Keep the first line under 72 characters
4. Create the commit.
5. Show the result with `git log --oneline -1`.
